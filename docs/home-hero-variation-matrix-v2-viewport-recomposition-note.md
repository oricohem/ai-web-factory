# Home Hero Finalist Viewport-Native Recomposition Note v2

## What was wrong with the previous normalization pass

- It improved polish, but several finalists still behaved like a centered module on a large background field.
- In multiple files, the real hero composition stayed constrained inside a contained rectangle while the viewport acted mostly as passive atmosphere.
- CTA/proof sometimes felt attached after the fact instead of being spatially integrated into a full-screen hero composition.
- Result: concepts were harder to judge as real homepage heroes, because they still read like expanded prototypes rather than viewport-native products.

## What changed in this recomposition pass (5 finalists only)

### 1) C5B — `c5b-floating-stage-object-theater.html`

- Rebuilt as edge-to-edge vertical stage composition with full-viewport participation.
- Anchor now occupies the majority of screen height with integrated stage layers, beams, and floor geometry.
- Copy sits in a deliberate top-left hero lane; proof and CTA are anchored to the same spatial system, not floating detached.

### 2) C9E — `c9e-productfirst-architectural-signature.html`

- Rebuilt as true product-first split where architectural anchor dominates the viewport width and height.
- Spatial object/architecture stack now uses full-screen scale and acts as primary visual reading.
- CTA and proof are integrated in the copy rail with rhythm aligned to anchor weight.

### 3) C8E — `c8e-framed-architectural-spotlight.html`

- Recomposed so framing is the actual concept structure across the viewport, not a card in a background.
- Frame rails and spotlight architecture now drive edge-to-edge layout and typography placement.
- Proof sits on the same composition grid and reads as built-in, not pasted.

### 4) C4B — `c4b-editorial-object-ledger.html`

- Rebuilt into a true editorial asymmetric screen architecture with strong left text spine and full-height right anchor world.
- Ledger rail + object stage now use viewport height and meaningful visual balance.
- CTA/proof now live in the editorial flow and remain readable at hero scale.

### 5) C5E — `c5e-floating-stage-architectural-atrium.html`

- Rebuilt from centered-circle module into full-screen atrium composition with dominant staged architecture.
- The atrium and stage floor now occupy a real viewport footprint; copy lane and proof are spatially integrated.
- Atmosphere now supports composition as structure, not only decorative background wash.

## Which finalists now truly use the viewport

Strong viewport-native pass:
- **C5B**
- **C9E**
- **C8E**
- **C4B**
- **C5E**

All five now avoid the previous centered-module-on-background failure mode and use full browser space as active composition.

## Best candidate for full homepage development

**Best candidate: C5B — Floating Stage Object Theater**

Why:
- Strongest viewport-native hierarchy: typography lane, dominant anchor mass, integrated action/proof zones.
- Most convincing premium product presence without dashboard/admin cues.
- Clear conversion path with high memorability and commercial confidence.

## Finalist to reject if still not strong enough

**Reject first if forced to narrow further: C8E**

Why:
- It is significantly improved and now viewport-native, but still relies on framing language that can drift toward showcase aesthetics if not tightly managed.
- Compared to C5B/C9E/C4B, it carries slightly higher risk of feeling presentation-led over product-led in later expansion.
