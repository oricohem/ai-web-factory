# Home Hero Taste-Scored Directions v1 (AI Web Factory)

## Scope and guardrails

- Deliverable type: docs + standalone prototypes only.
- Production app code intentionally untouched (`app/`, `components/`, routes, data, global styles, auth/backend/env/dependencies unchanged).
- Goal: break the repeated dark-dashboard/card-wall loop and propose materially different landing-grade homepage hero directions.

## Direction brief (before coding prototypes)

- Named direction set:
  1. `AI Production Core`
  2. `Premium Asset Factory`
  3. `Luminous Build Field`
- First-read goal: communicate "premium AI production system" before any dashboard density.
- Palette roles:
  - Direction 1: cool graphite + signal blue for systems credibility.
  - Direction 2: warm industrial neutrals + amber for premium production authority.
  - Direction 3: luminous structured field + deep navy CTA for modern commercial landing clarity.
- Typography hierarchy: large display headline first, supportive paragraph second, clear primary/secondary CTA pair third, proof artifact last.
- Anti-patterns intentionally avoided: sidebar-first admin layout, dashboard wall as hero, random glow on black, retro control room skins, generic pastel SaaS template.

## 1) AI Production Core

Prototype: `docs/home-hero-directions/ai-production-core.html`

### Visual anchor
- Central "production core" system sphere with structured rings, linked nodes, and controlled motion to communicate a live AI production engine (not decorative particles).

### Hero composition
- Left: large headline, concise product narrative, strong CTA pair.
- Right: dominant core anchor + compact "Live Product Proof" panel as supporting element.

### Why it feels modern
- Hero-first composition, reduced chrome, large display typography, single dominant visual system instead of box-heavy UI.

### Why it feels premium
- Restrained palette, disciplined contrast, controlled grid substrate, and focused motion that suggests engineered software rather than visual noise.

### Brand memory driver
- The centered "AI production core" becomes ownable and repeatable as a product signature.

### Adapted reference principles
- "One memorable centerpiece" from cinematic AI hero patterns.
- "Proof artifact as support" from premium product landing structures.
- "Atmosphere with structure" via subtle grid substrate and tonal depth.

### Intentionally not copied
- No recognizable competitor layout motifs.
- No random neon dots, no gaming HUD treatment, no control-room cliches.

### Risks
- Could drift into familiar dark-tech language if overused in implementation.
- Ring/node motif needs careful refinement to avoid visual cliche in production.

### Implementation notes
- In product implementation, treat the core as a reusable hero system component with reduced ambient effects on mobile and stricter contrast tuning.

## 2) Premium Asset Factory

Prototype: `docs/home-hero-directions/premium-asset-factory.html`

### Visual anchor
- Layered "production slab" assembly field: material planes, structural beams, and stamped quality surface treatment.

### Hero composition
- Left: assertive value proposition + dual CTA hierarchy.
- Right: large industrial premium anchor zone with integrated, minimal proof metrics at the bottom.

### Why it feels modern
- Avoids legacy factory literalism and dashboard-first patterns; uses compositional depth and editorial spacing with fewer containers.

### Why it feels premium
- Warm industrial palette, material hierarchy, controlled surfaces, and high-confidence whitespace give it "serious production brand" credibility.

### Brand memory driver
- Distinctive "certified production surface" framing and slab-stack geometry establish ownable visual DNA.

### Adapted reference principles
- "Material depth and confident spacing" from premium industrial/architectural references.
- "Hero sells the product before metrics" from landing-grade composition principles.

### Intentionally not copied
- No literal factories, machinery, vehicles, interiors, or retro command bay visuals.
- No dashboard card-wall framing.

### Risks
- If implementation pushes texture and framing too far, it can become heavy.
- Warm industrial palette must preserve accessibility and not read as warning-state UI.

### Implementation notes
- Keep this direction restrained in production: use slab language mainly in the hero + key proof block, not across all route surfaces.

## 3) Luminous Build Field

Prototype: `docs/home-hero-directions/luminous-build-field.html`

### Visual anchor
- Large atmospheric, multi-source luminous field with one compact proof artifact card and subtle orbital ring accent.

### Hero composition
- Left-weighted bold headline and CTA stack.
- Right supporting proof artifact module; atmosphere does most of the emotional framing.

### Why it feels modern
- Current AI product-landing posture: strong typography, clear CTA hierarchy, minimal artifact, and large contextual field.

### Why it feels premium
- Controlled tonal layering, calmer contrast, and reduced UI clutter make it feel productized and commercially mature.

### Brand memory driver
- Bright but structured build-field atmosphere offers a distinctive first impression compared with dark dashboard defaults.

### Adapted reference principles
- "Large atmospheric color field + minimal proof artifact" from modern AI landing principles.
- "Fewer boxes, more composition" from taste-engine anti-density guidance.

### Intentionally not copied
- Avoided generic pastel SaaS hero blocks by using stronger contrast anchors and operational proof language.
- No decorative blob-only hero.

### Risks
- Closest to broader market trend; can become generic if copy/visual system is not uniquely branded.
- Requires stricter identity elements in production (iconography, type treatment, branded motion behavior).

### Implementation notes
- Best used when paired with strong brand signature components; otherwise can converge toward polished-but-familiar SaaS.

## Recommended direction

Primary recommendation: **Premium Asset Factory (8.9)**.

Why:
- Best balance of premium credibility, memorability, and productized "AI production company" feel.
- Strongly breaks from dashboard/admin-tool defaults without becoming abstract marketing-only art direction.

Secondary option:
- **AI Production Core (8.7)** if the team wants a more explicit "AI system engine" metaphor.

## Honest self-critique

- Direction 2 is strongest overall, but still needs careful production constraints to avoid over-materialization.
- Direction 1 has excellent product relevance but may need additional originality work in final implementation to avoid familiar "AI-core" symbolism.
- Direction 3 is highly modern and commercially viable, but most vulnerable to generic SaaS convergence unless brand-specific identity elements are amplified.

- Not failed.
