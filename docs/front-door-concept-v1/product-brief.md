# AI Web Factory Front Door Concept v1 — Product Brief

## Product definition

AI Web Factory Front Door is a guided business intake experience for owners and operators who need a launch-ready system, not only a website mockup.

The core promise:

- AI Web Factory asks better business and market questions first.
- It generates a structured Launch Brief.
- An agent uses that brief to produce a complete launch kit:
  - website
  - copy pack
  - offer and pricing structure
  - campaign angles
  - email/social/ad assets
  - launch plan
  - QA/critic review
  - next-step recommendations

## Target users

1. Founder with an idea but no clear market narrative.
2. New business owner preparing first launch.
3. Existing business owner relaunching offer/site.
4. Agency operator needing repeatable client intake and handoff.
5. Campaign team needing fast launch systems (not one-off pages).

## Core user problem

Prompt-only website builders fail when business fundamentals are unclear:

- weak positioning
- vague audience
- mismatched offer and conversion goal
- random visual style selection
- no campaign-ready output beyond page content

Result: polished pages with weak commercial outcomes.

## Why this is different from prompt-only builders

### Prompt-only model

- Input quality is unpredictable.
- User must already know strategy.
- Output is usually page-first and asset-thin.
- Strategy, offer, and launch channels are under-specified.

### AI Web Factory model

- Intake system captures business signals before generation.
- Structured Launch Brief acts as strategy memory for agent execution.
- Output package is multi-asset and launch-oriented.
- QA/critic layer improves trust and decision confidence.

## Product positioning statement

AI Web Factory is a launch-kit studio that turns structured business intake into a complete launch system, so owners get clearer positioning and better conversion assets than prompt-only website generation.

## Front-door concept goals

1. Make the distinction obvious: "launch kit" vs "website builder."
2. Increase brief quality before generation.
3. Reduce poor-fit requests by forcing strategic choices.
4. Show users exactly what will be generated next.
5. Build confidence through transparent brief preview and output scope.

## Direction shaping (required non-trivial visual brief)

### Chosen direction: Brief-to-Launch Operating Surface

- First-read goal: "This is a guided operating surface that converts business signals into a launch kit."
- Palette roles:
  - Base: cool neutral fog
  - Surface: elevated white/ink cards
  - Accent: signal teal for progress/intent
  - Support accent: copper for critical decision points
- Typography hierarchy:
  - high-contrast display headline for positioning
  - operational subheads for process comprehension
  - compact metadata labels for generated outputs
- Anti-patterns intentionally avoided:
  - generic SaaS gradient hero
  - dashboard wall
  - dark control-room mimicry
  - tiny card matrix with decorative noise

### Alternative direction explored: Signal Map Storyboard

- Strength: highly memorable "market signal map" metaphor.
- Weakness: risks becoming abstract before users understand practical outputs.
- Decision: rejected for v1 front door because clarity and conversion confidence matter more than atmosphere-first abstraction at intake step.

## Scope for this v1 artifact

- Docs + static prototype only.
- No production route/component changes.
- No dependency changes.
- No backend logic.

## Success criteria for this concept

1. User can explain in one sentence what AI Web Factory actually produces.
2. User can complete a structured brief with relevant business constraints.
3. User can preview generated brief fields before agent execution.
4. User can see package differences (Brief Builder vs Launch/Growth/Agency kit).
5. Experience feels premium and productized without becoming generic SaaS.

## Non-goals for v1

- Live account/authentication flow.
- Real persistence or backend submission.
- Production migration.
- Real pricing commitments.
