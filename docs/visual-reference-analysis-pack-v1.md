# AI Web Factory Visual Reference Analysis Pack v1

## 1) Purpose

Pure Visual Execution Mode removes prototype/documentation bias, but strong output still requires explicit visual input. This pack gives future agents a practical execution system so premium landing pages do not regress into generic SaaS patterns.

This pack enforces four required inputs before coding:

- concrete references (principles, not copying)
- anti-pattern blacklist (what is forbidden)
- memorable visual moment requirement (exactly one intentional signature)
- observable visual acceptance criteria (pass/fail checks)

Use this pack for visual execution only. Do not copy external code, brands, layouts, or copyrighted assets. Extract reusable design principles and component patterns only.

## 2) Reference analysis framework

Use this framework to analyze any strong visual reference before implementation.

### Reference Analysis Card (required fields)

1. **Page type**
   - What is it: launch page, product page, studio page, OS-like product page, etc.
   - Conversion intent: signup, demo request, purchase, waitlist, contact.
2. **Hero composition**
   - Asymmetric/symmetric, text-to-artifact ratio, dominant focal element.
   - Entry sequence: what users read first, second, third.
3. **Typography scale**
   - Approximate desktop/mobile display size behavior.
   - Weight contrast and line-length discipline.
4. **Spacing rhythm**
   - Section spacing cadence (tight/medium/large alternation).
   - Breathing zones and compression zones.
5. **Color/accent usage**
   - Neutral base, accent role, accent frequency.
   - Where color is intentionally absent.
6. **Section rhythm**
   - Narrative order and alternation of section visual weight.
   - Dense vs minimal transitions.
7. **Visual anchor**
   - Single strongest artifact/moment that defines the page memory.
8. **CTA treatment**
   - Primary CTA placement, style, and contrast behavior.
   - Secondary CTA hierarchy and restraint.
9. **Proof/trust treatment**
   - Logos, testimonials, quantified outcomes, product evidence.
   - Trust location in the flow and visual prominence.
10. **Component richness**
    - Which sections are fully designed components vs text blocks.
    - Interaction states (hover, active, focus, scroll reveals).
11. **Responsive behavior**
    - What collapses, what preserves hierarchy, what gets simplified.
12. **What makes it memorable**
    - One sentence: exact compositional or interaction reason.
13. **What not to copy**
    - Explicitly list brand-specific motifs, exact layout signatures, protected imagery, and unique copy patterns to avoid.

### Output format (recommended)

For each reference, keep analysis to one short page with direct implementation notes:

- `Keep`: transferable principles
- `Adapt`: how to map principles to this business
- `Avoid`: no-copy constraints and anti-pattern triggers

## 3) Reference archetypes

### A) Editorial Premium SaaS

**When to use it**
- Product needs trust, clarity, and premium restraint (B2B SaaS, platform launch, enterprise-facing products).

**Visual personality**
- Calm, precise, editorial, high-confidence.

**Typography behavior**
- Large left-aligned display headline.
- Tight headline tracking, moderate body width, high heading/body contrast.

**Layout behavior**
- Asymmetric hero, generous whitespace, clear vertical rhythm.
- Mixed section weights (one sparse, one denser).

**Color/accent behavior**
- Neutral foundation, one controlled accent.
- Accent appears in CTA + key highlights only.

**Common components**
- Editorial hero with product proof artifact, customer logo strip, structured proof cards, comparison module.

**Memorable moment ideas**
- Split-hero artifact with layered product snapshot and annotated callouts.
- Type-led hero where one product term is treated as a visual object.

**Risks**
- Can become too quiet or too safe.

**Anti-patterns**
- Generic center hero with equal-weight buttons.
- Endless muted cards with no focal visual anchor.

### B) Bold Conversion Product

**When to use it**
- Product needs urgency and direct conversion momentum (launches, campaign pages, conversion-first SaaS).

**Visual personality**
- Direct, high-contrast, action-oriented.

**Typography behavior**
- Big confident display type with short, specific lines.
- Strong CTA-adjacent headline framing.

**Layout behavior**
- Hero starts with immediate offer clarity, then rapid proof.
- Intentional hard transitions between key sections.

**Color/accent behavior**
- Strong accent used sparingly but decisively on conversion moments.
- Contrast-based emphasis over decorative gradients.

**Common components**
- Conversion hero, quantified proof row, objection-handling FAQ, pricing preview, designed final CTA block.

**Memorable moment ideas**
- Conversion calculator card that animates a business-specific payoff.
- Product benchmark strip that morphs from “before” to “after.”

**Risks**
- Can become shouty or template-like.

**Anti-patterns**
- Salesy headline with vague verbs and zero product specificity.
- Repetitive promo banners without true product evidence.

### C) Spatial Product Artifact

**When to use it**
- Product is technically complex and benefits from a visualized “system artifact” story.

**Visual personality**
- Dimensional, product-centric, crafted.

**Typography behavior**
- Fewer words, stronger headline-object relationship.
- Supporting text explains artifact meaning, not fluff.

**Layout behavior**
- Hero anchored by one spatial artifact; supporting sections decode it.
- Alternates immersive sections with plain proof sections.

**Color/accent behavior**
- Restrained palette with depth cues.
- Accent tied to product states, not decoration.

**Common components**
- Spatial hero artifact, layered process sequence, high-fidelity product cutaways, structured trust row.

**Memorable moment ideas**
- Scroll-reactive product map that reveals layers of value.
- Multi-plane mockup stack that clarifies pipeline stages.

**Risks**
- Can drift into abstract tech art with weak conversion intent.

**Anti-patterns**
- Decorative 3D blob with no product explanation.
- Effects-heavy hero that obscures CTA clarity.

### D) Boutique Service Studio

**When to use it**
- High-touch service or hybrid product/service where craft and trust must feel personal and premium.

**Visual personality**
- Boutique, curated, intentional.

**Typography behavior**
- Strong typographic hierarchy with refined pacing.
- Sentence rhythm feels authored, not generated.

**Layout behavior**
- Narrative sequence: positioning -> process -> outcomes -> trust -> conversion.
- Rich section identities; no repeated templates.

**Color/accent behavior**
- Subtle tonal palette; accent introduced in key trust/conversion moments.

**Common components**
- Signature hero statement, process rail, selected outcomes grid, testimonial artifact, premium final CTA panel.

**Memorable moment ideas**
- Curated “work specimen” strip with interactive reveal.
- Process section that visually transforms from raw to refined state.

**Risks**
- Can become portfolio-like without clear offer.

**Anti-patterns**
- Overly poetic copy with unclear services.
- Generic agency section labels and placeholders.

### E) High-Density Product OS

**When to use it**
- Product value is tied to operational depth and control, but page must still sell premium clarity.

**Visual personality**
- Precise, robust, system-level confidence.

**Typography behavior**
- Clear hierarchy with compact but readable supporting text.
- Numerical content gets disciplined alignment.

**Layout behavior**
- Structured grid with selective asymmetry.
- Dense proof modules balanced by low-density breathing sections.

**Color/accent behavior**
- Neutral system base, minimal accent used for status and key controls.

**Common components**
- Product OS hero module, workflow matrix, data snapshot components, reliability/security trust block, conversion rail.

**Memorable moment ideas**
- Live-state control panel artifact tied to real business outcomes.
- Dense operational map that unfolds into simplified mobile narrative.

**Risks**
- Can feel like an internal admin panel.

**Anti-patterns**
- Card-wall dashboards as the entire page identity.
- Equal-weight metrics with no conversion storyline.

## 4) Specific anti-pattern blacklist

Future agents must actively avoid these concrete anti-patterns during premium landing-page execution:

1. No centered hero with generic subheadline + two buttons unless explicitly chosen for business reasons.
2. No three equal feature cards as the main value section.
3. No section labels like “Features” or “How it works” without product-specific naming.
4. No default blue/purple/green gradient as the main brand idea.
5. No icon-in-colored-circle system as the primary visual language.
6. No page built mostly from cards + paragraphs without designed artifacts.
7. No Tailwind-default card/shadow feel for core sections.
8. No perfectly equal spacing rhythm across every section.
9. No CTA label “Get started” unless business context requires that exact action.
10. No generic AI verbs (“elevate,” “transform,” “unlock”) as headline anchors.
11. No hero where the only visual is a blurred gradient blob.
12. No repeated identical section templates three or more times in a row.
13. No full page of dark surfaces with only weak glow differentiation.
14. No nav + hero + logo strip + card grid + FAQ in untouched template order when a stronger narrative order is available.
15. No metrics row with arbitrary numbers lacking context or unit.
16. No equal button hierarchy where primary and secondary CTAs look visually equivalent.
17. No final CTA treated as a plain text block afterthought.
18. No paragraph blocks wider than readable measure on desktop.
19. No entire page built from centered text blocks.
20. No “AI-generated” style stock iconography as proof content.
21. No trust section that appears only at the bottom if trust is critical early.
22. No decorative motion that does not reinforce product meaning.
23. No overuse of glassmorphism/backdrop blur as universal component styling.
24. No monochrome page with accent color applied indiscriminately to random labels.
25. No mobile layout that merely shrinks desktop composition without hierarchy redesign.
26. No body copy that could fit any AI startup unchanged.
27. No repeated button styles with no state differentiation (`hover`, `active`, `focus-visible`).
28. No giant hero text without corresponding proof artifact.
29. No proof claims without adjacent visual evidence component.
30. No side-by-side sections where both columns have equal visual weight in every case.
31. No product screenshot dropped into a generic laptop mockup as the only hero idea.
32. No section dividers that rely only on subtle gray lines when stronger rhythm contrast is needed.
33. No same-radius/same-shadow treatment across all blocks.
34. No one-size-fits-all testimonial card style detached from brand direction.
35. No “Request demo” and “Learn more” as default CTA pair without business-specific CTA language.
36. No landing page whose visual system reads as internal docs, backoffice tool, or prototype.

## 5) Memorable visual moment requirement

### Mandatory rule

Every North Star or premium visual execution page must contain **exactly one intentional memorable visual moment** that a designer would screenshot.

Before coding, the agent must declare:

1. what the moment is
2. where it appears
3. why it is specific to the business
4. how it avoids generic SaaS aesthetics
5. what CSS/component mechanism creates it

If this declaration is missing, visual execution is not ready.

### Ten example memorable moments

1. **Outcome Morph Hero**: Hero artifact morphs from “before state” to “after state” using business-specific metrics and a controlled transform sequence.
2. **Pipeline Constellation**: A node map in hero where each node corresponds to a product workflow step and reveals concrete output previews.
3. **Artifact Split Reveal**: A diagonal split component revealing raw input vs refined product output on drag/hover.
4. **Proof Heatline**: A horizontal proof strip with live-style confidence bands tied to product quality checkpoints.
5. **Command-to-Result Lens**: A command panel that animates into a visual result board with explicit business taxonomy labels.
6. **Trust Stack Compression**: Logos + compliance + testimonial evidence compress into a single compact module as user scrolls into decision zone.
7. **Section Weight Pivot**: Mid-page transition where sparse editorial section pivots into dense operational component with a clear product narrative jump.
8. **Signature CTA Chamber**: Final CTA sits in a designed conversion chamber with tailored proof badges and one dominant action.
9. **Mobile-First Reveal Reorder**: On mobile, the core visual artifact appears before long copy and preserves the same narrative intent.
10. **Brand-Linked Accent Trigger**: Accent color appears only when key product state is shown, making color semantically meaningful instead of decorative.

## 6) Observable visual acceptance checklist (10-point)

Use this checklist as pass/fail observations after implementation:

1. **Hero composition**: Hero is asymmetric or includes a deliberate grid-breaking moment.
2. **Display confidence**: Desktop display type is visibly large and intentional, not body-sized.
3. **Accent discipline**: Accent color is limited to defined roles and does not flood the page.
4. **Section contrast**: At least two major sections clearly differ in visual weight.
5. **Component completeness**: Every major section contains a designed visual component, not text-only blocks.
6. **Memorable moment**: Exactly one screenshot-worthy memorable moment is present and intentional.
7. **Aesthetic classification**: Page does not read as docs/internal-tool/prototype aesthetic.
8. **Copy specificity**: Headlines and CTAs use product-specific language, not generic AI marketing copy.
9. **Responsive integrity**: Mobile layout preserves hierarchy and conversion intent, not simple shrinkage.
10. **Final conversion quality**: Final CTA is a designed conversion moment with hierarchy and supporting proof.

Pass target: 10/10 for North Star quality; 8/10 minimum for candidate direction.

## 7) HarborLane diagnostic: why “strong” can still be “revise”

Based on the HarborLane benchmark summary pattern, a page can score as strong and still be marked **revise** when it is good evidence of mode improvement but not yet a definitive North Star.

Typical reasons:

- It is strong enough to prove Pure Visual Execution Mode improves quality over prototype defaults.
- It is not strong enough to become the canonical visual reference for future builds.
- Hero likely lacks a truly iconic artifact that anchors memory in one glance.
- Interaction/signature moment is present but not distinctive enough to feel ownable.
- Reference input was likely too narrow, producing competent execution without standout originality.

Diagnostic interpretation:

- **Keep direction** if architecture and hierarchy are solid.
- **Revise direction** when the hero artifact and signature moment do not yet create a clear “screenshot memory.”

## 8) How future agents should use this pack

Required order before any North Star or premium visual execution build:

1. **Choose one reference archetype**
   - Select exactly one archetype from Section 3 as primary execution mode.
2. **Select 5–8 anti-patterns to actively avoid**
   - Choose from Section 4 based on risk profile of the page.
3. **Define the memorable visual moment before coding**
   - Complete the 5-field declaration from Section 5.
4. **Define section visual weights**
   - Mark each section as low, medium, or high visual weight before implementation.
5. **Build HTML/CSS**
   - Implement components with explicit hierarchy and conversion intent.
6. **Evaluate against the 10-point checklist**
   - Score pass/fail from Section 6 and revise until criteria are met.

This pack is execution-oriented. Use it to make concrete visual decisions, not to produce strategy essays.
