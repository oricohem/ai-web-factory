# Homepage Hero Premium Direction v1

Date: 2026-05-03

## Context and anti-repetition note

This pass targets only the homepage above-the-fold hero at `/`. It must move from "internal dashboard overview" toward "premium AI product landing entry" while preserving the AI Web Factory identity and not changing routes, data, auth, backend, or dependencies.

Anti-repetition note: avoid repeating dense panel grids, control-room framing, terminal cues, and black-canvas-plus-glow treatment. Shift to a conversion-oriented hero with strong narrative hierarchy, clear CTA intent, and one ownable proof artifact.

## 1) Relevant UI/UX Pro Max-style reasoning patterns

- Use explicit style taxonomy selection before styling (pick one primary style lane, then constrain decisions to it).
- Use product-fit palette logic (base/surface/text/accent/state roles) instead of decorative gradients.
- Use typography pairing logic for conversion hierarchy (display headline + utility body/meta rhythm).
- Use landing-page conversion structure (headline -> proof -> CTA -> trust cues) rather than dashboard-first card density.
- Run anti-pattern and pre-delivery checks (generic SaaS look, weak CTA hierarchy, noisy atmosphere, weak mobile collapse, missing accessibility contrast/focus support).

## 2) Chosen landing-page pattern

**Pattern:** Narrative-first hero with split composition:

- Left: brand story, headline, concise value proposition, CTA stack.
- Right: product proof artifact (live-quality snapshot style panel).
- Bottom of hero: compact trust/proof strip.

This keeps clear conversion hierarchy without copying the concept file 1:1.

## 3) Chosen visual style

**Primary direction:** Precision Ambient Lab  
**Secondary direction:** Editorial Machine Room

Why: Precision Ambient Lab gives engineered premium depth and product-fit material tone, while Editorial Machine Room adds stronger headline authority and intentional whitespace needed for landing-page energy.

## 4) Palette reasoning

- Base: tinted graphite/charcoal (not pure black) to preserve depth and reduce generic dark SaaS feel.
- Surfaces: narrow neutral family with luminance steps for hierarchy, not many unrelated panel colors.
- Accent: restrained signal cyan for modern AI clarity and proof highlights.
- CTA accent: disciplined warm amber for primary action emphasis aligned with existing factory identity.
- Status colors remain semantic (green success, red risk), separated from brand/CTA accents.
- Accent coverage stays limited so contrast and hierarchy remain deliberate.

## 5) Typography direction

- Keep Geist/Geist Mono stack to preserve product identity and implementation stability.
- Increase headline scale and tighten tracking for a premium landing first read.
- Keep supporting copy readable (max line length, relaxed leading).
- Reserve mono usage for technical proof labels/metrics only.
- Ensure mobile headline wraps into strong short lines instead of long weak paragraphs.

## 6) Hero composition

- Build a layered atmosphere system above the fold:
  1. substrate (graphite tonal field)
  2. structure (aperture/rails to frame content)
  3. lighting (directional top-left and right-falloff pools)
  4. texture (low-opacity grain/technical contour)
  5. content protection scrim behind text/artifact
- Use large visual breathing room and one dominant read.
- Place one premium product-proof artifact panel in hero so credibility appears immediately.

## 7) CTA hierarchy

- Primary CTA: strongest contrast and visual weight, action-oriented copy.
- Secondary CTA: lower-emphasis ghost style for exploration path.
- Tertiary trust cue: compact evidence line (quality/readiness/review signals) under CTAs.
- Keep one clear primary action, no competing equals.

## 8) What to avoid

- Retro dashboard/terminal/blueprint motifs.
- Dense boxy admin panel treatment in the hero.
- Control-room HUD cliches.
- Purple-pink AI gradients, random glow blobs, full-screen glassmorphism.
- Tiny text on dark surfaces, weak contrast, over-segmented card mosaics.
- Reusing existing dashboard module layout as hero.

## 9) Why this should feel more modern than older dashboard concepts

- It prioritizes conversion narrative and product positioning before operational density.
- It introduces ownable hero composition and artifact-led proof instead of dashboard blocks as first impression.
- It uses structured atmosphere and material logic (not generic dark gradients) to create premium depth.
- It improves typographic authority and CTA clarity for a sellable product entry experience.
- It keeps AI Web Factory identity, but reframes the homepage as a premium product surface rather than an internal admin console.
