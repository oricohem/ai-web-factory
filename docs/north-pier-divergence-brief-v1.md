# North Pier Floral Divergence Extraction Brief v1

## 1. Executive summary
- The current North Pier pass inherits Mira's ecommerce-first skeleton, so the page reads like a product drop with floral nouns swapped in.
- Hero composition appears to reuse the same editorial-copy-plus-artifact grammar, which keeps first-read behavior too close to Mira.
- Surface language stays in the same warm ivory + rounded shell system, so visual DNA is preserved instead of re-authored.
- Section cadence mirrors Mira's long-breath narrative blocks ending in proof/FAQ/final CTA, producing the same pacing memory.
- CTA logic remains "primary commerce action + softer secondary action" rather than local studio booking choreography.
- Trust is framed as curated proof cards instead of neighborhood-service confidence signals (hours, location, capacity, lead times).
- Offer architecture feels like a launch bundle rather than a florist studio operating system (services + workshops + availability + inquiry).

3. Same rounded card-shell geometry language for mid-page content blocks.
4. Same hero grammar: editorial eyebrow + two-line headline + short lead + dual CTA row.
5. Same asymmetric hero split where copy sits opposite a "designed object" panel.
6. Same CTA styling logic: filled primary pill + low-contrast text-link secondary.

9. Same material-mood prose style ("tactile, warm, ritual") leading copy tone.
10. Same proof block behavior using short numbered claims in a compact list.
11. Same FAQ accordion pattern placed in late-page before final conversion.
12. Same final CTA chamber behavior that repeats the hero promise and two CTAs.
13. Same footer treatment with short brand line + compact descriptor + legal line.
14. Same low-motion interaction profile (subtle lift/fade only) without service-state signals.

## 3. Hard bans for North Pier v2
1. Must not use a two-column hero with copy on one side and a single artifact/object panel on the other.
2. Must not use Mira-like warm-ivory full-page background as the primary surface system; at least two distinct tonal zones are required.
3. Must not use rounded-pill primary CTA plus text-link secondary CTA pairing in hero or final CTA.
4. Must not place FAQ immediately before final CTA; FAQ must appear before booking and trust closure.
5. Must not use numbered proof chips/list items (`01/02/03`) as the trust module format.
6. Must not use a "first drop" or bundle-style offer card as the main conversion block.
7. Must not reuse identical section spacing cadence across the full page; at least three rhythm shifts (dense/airy/dense) are required.
8. Must not keep all sections inside one repeated max-width container; at least one full-bleed section and one narrow text rail are required.
9. Must not use Mira vocabulary in headings or CTA copy (`ritual`, `first drop`, `collection`, `shop`).
10. Must not rely on beige-neutral palette dominance; botanical green or stem-dark must carry structural roles (not accent-only).
11. Must not end with a mirrored hero promise line; final CTA must be operational ("check openings", "request event date", etc.).
12. Must not use identical card radius, border, and shadow tokens across all blocks; at least two component families must be visibly distinct.

## 5. New visual direction options
**Direction A — Tidal Noticeboard**
- One-sentence concept: North Pier feels like a living studio board where bookings, stems, and seasonal offers are pinned in motion-ready layers.
- Hero grammar: full-bleed board composition with overlaid availability strip, not split hero copy/object.
- Palette logic: salt-paper base, stem green, tide-charcoal, rope tan, and one coral booking accent.
- Typography logic: utility-forward sans for operations + restrained serif only for short emotional lead-ins.
- Section rhythm logic: dense operational strip -> spacious editorial interlude -> dense booking/workshop modules.
- Interaction logic: tab-like toggles for workshop dates and service tiers; hover reveals operational detail instead of decorative lift.
- Why it is different from Mira: replaces boutique-object storytelling with local operations choreography and board-style modularity.

**Direction B — Harbor Ledger**
- One-sentence concept: the page reads like a handcrafted studio ledger mapping what is available this week and how to reserve it.
- Hero grammar: top utility bar (hours + next workshop seats) followed by linear headline and booking action row.
- Palette logic: parchment-light base, ink-black text, oxidized green dividers, marigold state markers.
- Typography logic: high-contrast serif headlines paired with tabular utility metadata and compact labels.
- Section rhythm logic: linear scroll with hard separators and narrow text rails instead of large floating blocks.
- Interaction logic: accordion-to-form transitions, inline seat counters, and explicit state chips (open/limited/full).
- Why it is different from Mira: abandons soft rounded ecommerce shells for rule-based editorial-service structure.

**Direction C — Greenhouse Run Sheet**
- One-sentence concept: North Pier presents services and workshops like a day-of production run sheet for a boutique floral studio.
- Hero grammar: immediate "this week at the studio" schedule card stack with concise headline above.
- Palette logic: fog white, greenhouse deep green, soil brown, petal blush, and alert amber for capacity states.
- Typography logic: condensed utility headers + warm body copy, prioritizing scan speed over poetic paragraphing.
- Section rhythm logic: operational blocks first, inspiration/proof second, booking closure third.
- Interaction logic: expandable schedule rows, date-filter chips, and sticky mini booking bar on scroll.
- Why it is different from Mira: prioritizes calendar and service operations over atmospheric product narrative.

## 6. Pick one direction
Recommended direction: **Harbor Ledger**.

Harbor Ledger is the strongest divergence because it replaces Mira's soft atmospheric ecommerce grammar with an unmistakable local-service information architecture: utility-first top bar, linear separators, operational metadata, and booking-state clarity. It still fits the approved references by combining Soil & Stem instructional clarity, Movement Lab's restraint, and Boston Tattoo's local trust immediacy, while avoiding copycat aesthetics from any single source.

## 7. New component recipe set
1. **Nav recipe — Local utility masthead:** left brand lockup, center quick links (`Services`, `Workshops`, `Availability`, `FAQ`), right micro-utility block (today hours + phone + "Book").
2. **Hero recipe — Availability-first opener:** single-column headline over a two-row operations band (next workshop date, seats left, lead time, primary booking action).
3. **Service/offer block recipe — Three service rails:** `Weekly Arrangements`, `Event Florals`, `Studio Pickup` displayed as asymmetric rows with deliverables, lead time, and reserve button.
4. **Workshop block recipe — Date ledger stack:** upcoming workshop cards in descending date order with level tag, seat count state, and inline "reserve seat" action.
5. **Proof/trust block recipe — Local confidence strip:** studio photos thumbnail row + address/hours/contact + "recent event partners" text list (no numbered proof chips).
6. **Booking/availability block recipe — Capacity matrix:** week-view grid showing open/limited/full states plus form entry point for custom requests.
7. **FAQ block recipe — Decision FAQ:** five short decision-friction questions grouped by `Booking`, `Pickup`, `Events`, each answer ending with next action.
8. **Final CTA/footer recipe — Action dock + grounded footer:** sticky-style final action dock ("Check openings", "Request event date") above a practical footer with map link, hours, and response window.
