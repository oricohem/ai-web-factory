# Mira North Star Learnings v1

## 1) Executive summary

Mira became the first successful visual candidate because the workflow changed from a broad prompt that produced generic output to a constrained sequence: narrow references -> focused brief -> selected iconic moment -> build -> narrow refinement.  
This process shift reduced style drift and forced one coherent visual family (`Warm Editorial Commerce`) tied to a specific product world instead of mixed-category aesthetics.

## 2) What worked

The following factors are clearly supported by the Mira source brief, build, and evaluation:

- **Narrow 3-reference brief:** Mira intentionally limited references to Little Karma Co, NOHMA, and Boston Tattoo Company, which prevented broad style blending.
- **One visual family:** The brief selected one family (`Warm Editorial Commerce`) instead of mixing multiple archetypes.
- **Product-specific background strategy:** The hero world was tied to ceramic/scent ritual semantics rather than decorative atmosphere.
- **Pre-selected iconic moment:** The process required options and then committed to one selected moment (Refill Ritual Cutaway).
- **Zero annotation constraint:** The hero object reads without callout/diagram annotation language, improving boutique plausibility.
- **Warmth proof:** Material cues (matte grain, cast shadow, shelf plane, warm surface tones) made the page feel tactile and believable.
- **Gentle interaction choice:** Interaction stayed restrained (subtle hover lift/haze behavior) and supported material perception instead of flashy effects.
- **One narrow refinement pass:** The evaluation confirms measurable gains after refinement in hero depth, typography hierarchy, and section rhythm.
- **Honest revise verdict:** The team kept the verdict as `revise`, which preserved evaluation integrity and prevented premature benchmark promotion.

## 3) What still did not fully work

Based on `evaluation.md`, Mira is a **candidate**, not a final benchmark:

- Supporting sections improved but still were not as iconic/memorable as the hero artifact.
- Overall North Star usefulness remained **8/10**, which is strong but not definitive benchmark quality.
- Typography and rhythm improved, but typography was still described as one refinement away from ownable/world-class signature quality.
- Success in this one category (boutique ecommerce product story) does not prove transfer across all target categories.

## 4) Repeatable workflow (7 steps)

### Step 5 — Choose category
Pick one business category first (for example boutique ecommerce or boutique local service).  
**Prevents:** abstract styling choices with no business-fit constraints.

### Step 6 — Select 3 references only
Use exactly three references that support the same intended family.  
**Prevents:** visual collage behavior, conflicting cues, and generic blended output.

### Step 7 — Create narrow build brief
Write one constrained brief with product/business specifics, required feel, and forbidden patterns.  
**Prevents:** prompt vagueness and uncontrolled interpretation spread.

### Step 8 — Propose 3 iconic moments
Generate three candidate screenshot-worthy moments tied to the business story.  
**Prevents:** entering build phase without explicit memory-anchor options.

### Step 9 — User selects one iconic moment
Wait for explicit selection before coding.  
**Prevents:** agent self-selection drift and mismatch with stakeholder taste.

### Step 10 — Build one page
Implement one complete page around the chosen moment and selected family.  
**Prevents:** multi-page scope creep and diluted visual concentration.

### Step 11 — Run one focused refinement pass
Execute one narrow revision pass against observed weaknesses from evaluation.  
**Prevents:** endless redesign loops and unfocused “change everything” iterations.

## 5) Early token extraction (candidate token principles)

These are **candidate token principles** extracted from Mira behavior, not final token sets.

- **Palette behavior:** Warm off-white/sand/clay neutrals dominate; dark charcoal drives text contrast; olive/amber stay rare and semantic.
- **Typography behavior:** High contrast between editorial display and compact utility labels; short line lengths and controlled tracking preserve calm premium tone.
- **Spacing behavior:** Large hero/section breathing zones with selective compression in proof/offer modules to maintain narrative rhythm.
- **Surface/card behavior:** Soft warm surfaces with subtle tonal gradients and rounded geometry; card treatment supports object story rather than dashboard density.
- **Shadow/depth behavior:** Warm, low-noise shadow stacks and inset highlights communicate tactile depth (especially ceramic object and shelf plane).
- **Motion/interaction behavior:** Minimal, slow, material-linked motion (hover lift/haze drift) reinforces object realism and avoids attention theft.

## 6) Component recipe candidates (from Mira)

### 1) Product-specific artifact hero
- **What it does:** Centers one product-tied visual artifact as page memory anchor.
- **When to use it:** When one object/process can carry brand story in first glance.
- **What made it work in Mira:** Refill cutaway clearly explained shell + insert relationship without diagrams.
- **How to adapt to another business:** Swap artifact semantics (service tool, food object, creator kit) while preserving single-anchor discipline.
- **Failure risk:** Can become decorative sculpture if the artifact is not tied to offer meaning.

### 2) Tactile material surface
- **What it does:** Builds believable material context via surface plane, grain, and warm depth.
- **When to use it:** Material-led products or any offer that benefits from physical credibility cues.
- **What made it work in Mira:** Shelf plane + cast shadow + ceramic grain established warmth proof.
- **How to adapt to another business:** Translate to relevant material language (linen, wood, steel, paper, stone) with restrained treatment.
- **Failure risk:** Over-texturing can reduce legibility and feel staged/artificial.

### 3) Editorial ecommerce product story
- **What it does:** Moves from hero memory to curated product narrative modules.
- **When to use it:** Boutique ecommerce and premium product storytelling flows.
- **What made it work in Mira:** Collection, ritual, and material sections stayed commerce-specific and avoided SaaS framing.
- **How to adapt to another business:** Replace product copy/modules with category-specific proof/story while keeping editorial pacing.
- **Failure risk:** Can collapse into generic catalog blocks if section naming/content loses specificity.

### 4) Quiet proof/trust module
- **What it does:** Adds credibility in a calm, boutique-compatible format.
- **When to use it:** When trust evidence is needed without enterprise dashboard style.
- **What made it work in Mira:** Numbered proof items supported reliability without noisy badges/charts.
- **How to adapt to another business:** Use small, verifiable trust units (batch size, turnaround, quality checks, local proof).
- **Failure risk:** Weak or vague proof text makes the module look like filler.

### 5) First-drop offer card
- **What it does:** Packages launch offer, pricing, and action into one conversion-focused module.
- **When to use it:** First-release or limited-run product drops.
- **What made it work in Mira:** Price, inclusion details, and shipping cadence were combined in a clear warm module.
- **How to adapt to another business:** Convert to category-equivalent offer framing (starter package, reservation set, first cohort).
- **Failure risk:** If hierarchy is weak, it reads like a generic pricing card rather than a designed offer moment.

### 6) Final boutique CTA band
- **What it does:** Closes page with a designed conversion chamber consistent with brand atmosphere.
- **When to use it:** Final decision zone where action and confidence need reinforcement.
- **What made it work in Mira:** Warm final band repeated core promise and dual intent CTAs in coherent tone.
- **How to adapt to another business:** Reframe headline/promise to category outcome while preserving designed closure.
- **Failure risk:** Can become an afterthought strip if visual weight and message specificity are too low.

## 7) Anti-patterns confirmed by Mira

Mira successfully avoided these failure modes:

- SaaS hero
- generic gradient
- annotation-heavy diagram
- startup cards
- B2B feature rhythm
- sterile catalog
- decorative-only background

## 8) North Star acceptance lesson

A page can be worth merging as a **candidate** even with verdict `revise` when it proves a workflow breakthrough that was previously missing.

### Status definitions

- **reject:** did not prove new visual capability.
- **candidate:** proves capability but needs more examples and stronger repeatability.
- **benchmark:** strong and transferable enough to become reusable standard.

### Mira classification

**Mira status: candidate.**  
Reason: it proved the constrained workflow can produce a North-Star-strength hero and stronger rhythm/tactility, but does not yet establish cross-category benchmark confidence.

## 9) Recommendation for next experiment

**Recommended next North Star category: Boutique local service.**

Why this should be next:

- It tests whether the same constrained workflow can transfer beyond ecommerce product-object storytelling.
- Existing reference process already includes local/boutique warmth cues, so it is a logical adjacent stress test.
- It will validate whether “one iconic moment + narrow refinement” can produce trust and personality in service conversion flow.

What should be different from Mira:

- Iconic moment should represent service credibility/experience (not a physical product cutaway).
- Proof modules should emphasize local trust signals and service outcomes over product material logic.
- CTA architecture should prioritize booking/contact intent while preserving boutique atmosphere.

## 10) Exact next-step instruction

Do not build another page until a narrow reference brief and 3 iconic moment options are created for the next category.
