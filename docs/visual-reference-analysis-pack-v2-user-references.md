# AI Web Factory Visual Reference Analysis Pack v2 (User-Selected References)

## 1) Executive summary

The selected references indicate a taste profile that values **business-specific visual worlds** over generic template styling. The user repeatedly prefers pages where the background, hero artifact, and atmosphere clearly belong to the business concept (not just a decorative gradient layer).

Key taste signals:

- **Business-specific backgrounds:** preferred when the hero world explains the offer at first glance (for example, product context, transformation context, or service atmosphere), not abstract ambiance.
- **Memorable visual anchors:** favored experiences include one dominant moment (3D object, artifact reveal, bold image-led hero, or distinctive material texture) that makes the page screenshot-worthy.
- **Boutique/local brand feel:** local-service references suggest preference for warmth, personality, editorial character, and human credibility over enterprise dashboard language.
- **Premium ecommerce polish:** ecommerce references suggest interest in tactile surfaces, product-first framing, and intentional composition that feels curated rather than catalog-default.
- **Selective 3D/motion:** 3D and animation are liked when they add concept clarity or emotional lift; they are not universal defaults.
- **Designed AI/SaaS pages:** AI references show preference for pages that feel authored and visually intentional, with strong background logic, not interchangeable neon-SaaS patterns.

Interpretation rule for future work: these references are **principle sources**, not category lock-ins. The core preference is **fit between visual system and business story**.

## 4) Design principles extracted from the user’s taste (16)

1. Backgrounds must be concept-specific, not decorative.
2. Every page needs one memorable visual anchor with clear business relevance.
3. Local/service pages should prioritize human trust and brand warmth over SaaS framing.
4. Ecommerce pages should create tactile product atmosphere, not bare catalog grids.
5. AI pages should feel designed and intentional, not interchangeable startup templates.
6. 3D/motion should be optional and justified by story clarity.
7. If 3D/motion is used, a static and reduced-motion fallback is mandatory.
8. Hero visuals should explain offer context in the first read.
9. Atmosphere should support hierarchy; text legibility always wins.
10. Premium polish comes from composition discipline, not effect density.
11. Pricing sections should function as decision support, not generic card triplets.
12. Proof modules should be integrated into the visual world, not appended as afterthoughts.
13. Visual personality should match business type while avoiding category stereotypes.
14. Detail-rich backgrounds require protected contrast zones for CTA and key copy.
15. Simpler page structures can be elevated by one strong signature moment.
16. Extract principles from references; never replicate branded assets, language, or layouts.

## 5) New anti-patterns based on these references (24)

1. No generic SaaS gradient unless tied to business concept semantics.
2. No abstract AI backdrop that could belong to any startup.
3. No 3D object unless it dramatizes or clarifies the product story.
4. No local business page styled like enterprise B2B software.
5. No ecommerce page without tactile or atmospheric product context.
6. No pricing table detached from brand tone and proof.
7. No hero where image/background has zero relationship to the offer.
8. No “AI neon cloud” visuals used as default identity.
9. No equal-weight section stacks with no signature focal point.
10. No motion that competes with reading order or CTA discovery.
11. No complex atmosphere directly behind dense body copy.
12. No card-wall primary section for boutique/local storytelling pages.
13. No template-order flow when a stronger business narrative is obvious.
14. No generic icon circles as the dominant visual language.
15. No proof strip that looks pasted from another brand style system.
16. No before/after service without transformation artifact evidence.
17. No local-service trust section hidden at the very bottom only.
18. No premium ecommerce concept with flat utility-only spacing rhythm.
19. No decorative particles/meshes without semantic role.
20. No “all sections same mood” pages; narrative contrast is required.
21. No over-detailed hero that hides first CTA above the fold.
22. No branded copy mimicry from references.
23. No recognizable layout cloning from any single reference.
24. No multi-reference collage that combines motifs without a coherent system.

## 7) How to use this in the next North Star build

Before any coding starts, the agent must complete this required pre-build choice set:

1. Choose **one visual family** from Section 2 as the primary direction.
2. Choose **one memorable visual moment** (single anchor, not multiple competing ideas).
3. Choose **five anti-patterns to actively avoid** from Section 5.
4. Choose **one background strategy**:
   - concept-specific product backdrop
   - warm editorial surface system
   - restrained AI atmosphere layers
   - minimal neutral with one high-impact artifact
5. Decide whether **3D/motion is appropriate or forbidden**:
   - if appropriate: define semantic role + fallback behavior
   - if forbidden: define static signature alternative
6. If pricing/proof is relevant, choose **one pricing/proof strategy**:
   - decision-support pricing system
   - transformation-evidence proof
   - trust-first service credibility narrative

Gate rule: if any of the six choices are missing, implementation is not ready.
