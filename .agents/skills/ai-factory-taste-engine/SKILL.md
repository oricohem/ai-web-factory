# AI Web Factory Taste Engine v1

## 1. Purpose

This skill is a project-local taste and quality engine for AI Web Factory visual work. It teaches future agents how to evaluate and create modern premium AI product visuals with product-fit judgment, not template defaults.

Use this skill before any non-trivial UI design, redesign, review, or polish task.

## 2. Taste anchors

Use references as principle sources, not copy targets. AI Web Factory should learn from:

- modern premium AI landing pages
- bold hero-first product sites
- cinematic but clean product pages
- visual centerpiece-driven homepages
- atmospheric industrial or premium brand sites
- minimal but memorable product experiences

Extract reusable principles:

- hierarchy strategy
- visual anchor strategy
- density control
- trust signals
- atmosphere discipline

Never copy layouts, branded assets, exact copy, or recognizable motifs one-to-one.

## 3. What good looks like

Strong modern premium work in this product shows:

- strong first-read hero
- large confident typography
- one memorable visual anchor
- clear CTA hierarchy
- atmospheric but controlled background
- product proof artifact
- generous spacing
- fewer boxes
- brand presence before dashboard density
- commercial trust
- modern responsive behavior

## 4. What bad looks like

Reject repeated low-quality patterns:

- generic dark dashboard
- weak glow on black
- sidebar-first admin layout
- card wall
- cards inside cards
- retro control room
- terminal/HUD/CAD/blueprint skin
- tiny cosmetic diffs
- fake enterprise metrics
- old internal tool feeling
- generic SaaS template with no identity

## 5. Reference extraction method

For each reference set, answer:

1. What is the visual anchor?
2. What is the hero composition?
3. What makes it feel modern?
4. What makes it feel premium?
5. What gives it brand memory?
6. What should be adapted?
7. What must not be copied?
8. What would become generic if used blindly?

Keep notes short and actionable. Focus on transferable principles for AI Web Factory.

## 6. Design scoring rubric

Score every candidate direction across 10 dimensions, each from 0 to 1:

1. First-read clarity
2. Visual memorability
3. Modernity
4. Premium feel
5. Product relevance
6. Typography hierarchy
7. Background/atmosphere quality
8. CTA clarity
9. Layout originality
10. Commercial trust

Total score range: 0.0 to 10.0.

Decision thresholds:

- below 7.0 = reject
- 7.0 to 8.0 = usable but not enough
- 8.5+ = acceptable foundation
- 9.0+ = strong direction worth expanding

## 7. AI Web Factory-specific target

AI Web Factory should feel like a premium AI production system for creating, reviewing, and shipping high-quality digital assets, not an internal dashboard.

Target feel:

- modern
- premium
- useful
- commercially credible
- productized
- visually memorable
- AI-native
- built for high-quality output

## 8. Required visual concept process

Before production UI work:

1. Produce 2 to 3 materially different directions.
2. Explain the visual anchor for each direction.
3. Score each direction using the rubric.
4. Reject weak directions.
5. Choose one direction.
6. Only then implement a focused pilot.

Materially different means meaningful changes in composition, hierarchy, and atmosphere, not palette-only or tiny style tweaks.

## 9. Implementation guardrails

When implementing:

- start with one route or one hero zone
- do not redesign the entire app at once
- do not add broad CSS without visible effect
- do not use dashboard density as the hero
- dashboard/proof artifacts may appear, but only as supporting elements
- avoid left-sidebar admin layout on marketing or landing homepages unless explicitly required

## 10. Completion checklist

Future agents cannot mark visual work complete unless all are true:

- screenshots prove meaningful visual difference
- desktop and mobile are checked
- design score is included
- self-critique is included
- anti-patterns are checked
- the result is not just a small variation of previous attempts

## 11. Reviewer checklist

Human PR reviewers can use this quick gate:

- Does this feel like a premium product?
- Is there a memorable visual anchor?
- Is it meaningfully different from the previous state?
- Would a customer trust and pay for this?
- Is the dashboard only supporting proof, not the entire design?
- Did the agent score the result honestly?

## 12. Next recommended workflow

For the next UI implementation pass, use:

Reference-informed, taste-engine-scored, one-route pilot only.

## 13. Concrete taste anchors and translation rules

Future agents should learn from these reference qualities and apply them with product-fit judgment.

### 13.1 Modern gradient AI product landing pages

Take:

- bold simple hero
- clear CTA
- large atmospheric color field
- minimal proof artifact

Do not take:

- generic pastel SaaS sameness
- empty marketing copy

### 13.2 Cinematic particle or AI-core hero pages

Take:

- one memorable visual centerpiece
- depth
- restrained text
- strong focus

Do not take:

- sci-fi cliche
- random glowing dots
- gaming neon

### 13.3 Premium industrial or architectural brand sites

Take:

- material depth
- confident spacing
- strong atmosphere
- brand presence

Do not take:

- literal vehicles, interiors, or unrelated imagery

### 13.4 Dark sculptural agency heroes

Take:

- large typography over a tactile dimensional background

Do not take:

- generic agency purple
- decorative waves without product meaning

### 13.5 Photo-led hero systems

Take:

- human-scale emotional impact
- cinematic lighting

Do not take:

- random stock-photo decoration unless the image directly supports the product story

### 13.6 AI Web Factory translation rules

Translate all reference learning into:

- a signature AI production visual anchor
- a hero that sells the product before showing the dashboard
- dashboard/proof UI as a supporting artifact only
- fewer boxes and more composition
- memorable background system with product meaning
