# AI Factory Design Research Corpus Skill

Use this skill before major AI Web Factory website generation work, full-site visual direction work, broad landing-page concepting, or any design task where agents need more than the current internal dark geometric product-object style family.

This is a research and reasoning skill. It does not authorize copying websites, branded assets, screenshots, layouts, motion systems, copy, or copyrighted content.

## Required source

Read `docs/design-research-corpus-v1.md` before shaping directions.

Also use:

- `ai-factory-taste-engine` for taste scoring and anti-pattern checks.
- `ai-factory-design-intelligence` for AI Web Factory product fit.
- `docs/design-learning-report-v1.md` for the anti-loop baseline.
- `docs/design-skills.md` and `AGENTS.md` for QA requirements.

## When to use

Use this skill for:

- full website visual generation
- homepage or landing-page art direction
- major hero direction work
- reference-informed design strategy
- broad visual identity exploration
- avoiding same-family repeated concepts
- creating direction briefs before implementation

Do not use it as a reason to modify production UI when the task is docs-only or research-only.

## Research constraints

- Use web research when available, but extract principles only.
- Do not scrape assets.
- Do not copy branded visual systems.
- Do not reproduce screenshots, copyrighted content, or recognizable layouts.
- Do not import reference copy, metrics, logos, testimonials, or badges.
- Transform every reference principle into AI Web Factory's product story.

## Required workflow

Before implementation, produce a concise direction brief:

1. Define product positioning.
2. Choose target audience.
3. Select 2-3 reference categories from `docs/design-research-corpus-v1.md`.
4. Select at least 8 design genes.
5. Explain the gene combination.
6. Generate 3 materially different full-site directions.
7. Score each direction with the Taste Engine and the corpus scoring expansion.
8. Reject same-family repeats.
9. Choose one direction only after critique.
10. Define homepage structure, section flow, responsive behavior, and proof/QA plan.

## Reference-to-originality rule

For every reference principle used, transform it across at least 3 dimensions:

1. Different product story.
2. Different layout structure.
3. Different visual anchor or material treatment.

Reject a direction if it can be recognized as a specific reference after swapping the logo.

## Design gene minimum

Every major direction must choose at least 8 genes from the corpus, including:

- composition family
- visual anchor type
- background system
- typography rhythm
- proof strategy
- CTA strategy
- section flow
- responsive behavior pattern

Add more genes when useful: content density, brand mood, motion potential, color strategy, material texture, spatial depth, and product artifact style.

## Anti-repetition gate

Reject directions that default to:

- generic SaaS gradients
- random 3D blobs
- meaningless particles
- stock-photo decoration without product meaning
- dashboard-first hero
- card wall
- dark admin UI
- control room / HUD / blueprint repetition
- copying reference layouts too closely
- one-style repetition across multiple attempts
- decorative backgrounds that do not support product meaning

## Output format

Use this format before coding:

- Product positioning
- Target audience
- Selected reference categories
- Selected design genes
- Three full-site direction briefs
- Reference-to-originality transformations
- Taste Engine score
- Corpus scoring expansion
- Rejected same-family repeats
- Chosen direction and rationale
- Homepage structure
- Responsive strategy
- Proof and QA plan

## Completion rule

For docs-only research tasks, no screenshots, desktop walkthroughs, Playwright captures, or video are required unless explicitly requested.

For production UI implementation, follow the QA and screenshot requirements in `AGENTS.md`, `docs/design-skills.md`, and the relevant AI Web Factory design skills.
